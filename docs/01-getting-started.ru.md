# Начало работы

> Подробный разбор. В [README](../README.ru.md) есть сжатый туториал; эта страница раскрывает требования,
> два способа подключения и установку. Английская версия — [01-getting-started.md](01-getting-started.md).

## Требования — прочитайте сначала

tms-pipeline настраивает методологию поставки **поверх существующего проекта**. Прежде чем начать, у вас
должны быть:

- репозиторий с настоящим исходным кодом;
- база документации (дерево docs, вики, хранилище Obsidian — где угодно, где живут долговечные знания о
  продукте/архитектуре);
- в идеале — бэклог задач.

Это **не** генератор проектов. Он не будет придумывать функции и не соберёт приложение из ничего. Если у
вашего проекта пока нет документации или бэклога, включённые пустые скелеты (`templates/docs-vault/`) дают
стартовую структуру, но наполнение их реальными продуктовыми решениями — ваша работа, выполняемая по-вашему.

## Два способа подключения

Оба бесплатны и используют одно и то же расцепленное ядро.

### A. «Под ключ» — настройка через мастер

Чтобы быстро запуститься на вашем проекте:

```bash
# 1. Онбординг проекта (пишет AGENTS.md + .claude/CLAUDE.md, по желанию кладёт скелеты)
npx tms-pipeline

# 2. Установить скиллы/агентов в Claude Code
/plugin marketplace add TmsNine/tms-pipeline
/plugin install tms-pipeline@tms-pipeline
/reload-plugins
```

Мастер `npx` задаёт короткий список вопросов (нажмите Enter, чтобы принять каждое значение по умолчанию) и
записывает заполненные `AGENTS.md` и `.claude/CLAUDE.md` в ваш проект. Также можно запустить онбординг прямо
из Claude Code командой `/tms-init`.

### B. «Методология» — подключить вручную

Если вы предпочитаете понять и подключить всё самостоятельно:

1. Прочитайте [методологию](00-methodology.ru.md).
2. Установите плагин (две команды `/plugin` выше) или скопируйте папки `skills/` и `agents/` в ваш
   `~/.claude/` (и `~/.codex/skills/` для Codex).
3. Скопируйте `templates/AGENTS.template.md` в ваш репозиторий как `AGENTS.md` и заполните плейсхолдеры
   вручную (см. [конфигурацию](02-configuration.ru.md)).

## Проверка установки

В Claude Code выполните `/help` или откройте список навыков — вы должны увидеть навыки `tms-*`. Затем
запустите задачу (см. туториал в README).

## Codex

Codex читает `AGENTS.md` нативно, поэтому методология применяется так же. Тела навыков не зависят от
инструмента. У Codex нет аналога `/plugin install`, поэтому скиллы и агентов нужно положить туда, где
Codex их ищет — в `~/.codex`:

- проще всего: запустите `npx tms-pipeline`, выберите Codex и согласитесь на копирование — мастер сам
  скопирует `skills/` → `~/.codex/skills/` и `agents/` → `~/.codex/agents/`;
- вручную: `cp -R skills/* ~/.codex/skills/ && cp -R agents/* ~/.codex/agents/`.

Различия в каталогах см. в [конфигурации](02-configuration.ru.md#codex).
